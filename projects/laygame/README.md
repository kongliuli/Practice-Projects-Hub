# laygame - 自动化卡牌游戏逻辑与模拟

## 核心需求
实现一个卡牌游戏的核心逻辑，包括但不限于：
1.  **角色属性系统**：定义基础属性（力量 Strength、敏捷 Agility、智力 Intelligence）及其派生属性（攻击 Attack、血量 Health、防御 Defense）。
2.  **技能系统**：支持主动技能 (ProactiveSkill) 和被动技能 (PassiveSkill)，并定义技能等级和使用类型。
3.  **卡牌实体**：将角色属性和技能封装为卡牌实体 (RoleCard)，包含最大血量、当前血量、攻击范围等。

## 开发目标
-   能够模拟卡牌对战过程，验证游戏平衡性。
-   作为游戏 AI 的基础，实现简单的 AI 对战逻辑。
-   提供一个基础的 WinForms UI 界面，用于展示卡牌信息和对战过程。

## 建议技术栈
-   **语言**: C#
-   **框架**: .NET 8 (或更高版本)
-   **UI**: WinForms (用于基础展示，可考虑升级至 WPF/Blazor 进行重构)
-   **数据存储**: 内存或轻量级文件存储 (如 JSON)，用于卡牌和技能数据。

## Solo 模式开发提示
-   **指令示例**: "基于现有 Role.cs 和 Skills.cs，实现一个简单的卡牌对战模拟器，展示两张卡牌的战斗过程。"
-   **关注点**: 优先实现核心战斗逻辑，UI 可简化。
